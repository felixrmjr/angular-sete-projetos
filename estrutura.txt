|- PASTA: e2e           => Diretorio voltado para teste de integração
|- PASTA: node_modulos  => Diretorio das bibliotecas do projeto, sempre alterado via comando NPM
|- PASTA: src           => Diretorio do codigo da aplicação
    |- app                  => Diretorio do codigo da aplicação
        | - app.component.css           => Arquivo de CSS do especifico do componente
        | - app.component.html          => Arquivo HTML com o que vai renderizar na tela
        | - app.component.specs.ts      => Arquivo de teste especifico do componente
        | - app.component.ts            => Arquivo do componente
        | - app.module.ts               => Arquivo do modulo que contem o componente
    |- assets               => Diretorio de imagens e arquivos estaticos (IMAGENS, CSS)
    |- environments         => Diretorio para criar versão de produção e desenvolvimento da aplicação
    |- index.htmnl          => Arquivo principal, toda pagina é carragado atraves do index
    |- main.ts              => Arquivo de inicialização do angular, que inicializa o modulo padrão
    |- polyfills.ts         => Adicionar bibliotecas externas ao angular CLI
    |- styles.css           => Arquivo global de css 
|- .angula-cli.json     => Arquivo de configuração do angular CLI
|- .editorconfig        => Configuração do editorconfig
|- karma.conf.json      => Configuração de teste unitário
|- package.json         => Arquivo principal de configuração node
|- protractor.conf      => Arquivo de testes de integração do angular
|- tsconfig.json        => Informações sobre a compilação
|- tslint.json          => Validador de codigo
